# 记忆翻牌小游戏（全栈版）PRD

## 1. 背景与目标

做一个可直接上线的现代化记忆翻牌小游戏，提供流畅交互与清晰反馈，并支持“无密码用户名登录 + 成绩上榜”的社交玩法，适合轻量娱乐与前端/全栈练手。

**产品目标**
- 玩家 30 秒内理解玩法并开始游玩
- 完整一局体验顺畅（翻牌、结算、胜利、再来一局）
- 登录用户可自动提交成绩并查看排行榜
- 可部署到 Vercel 线上运行

## 2. 范围说明

**本期范围（MVP）**
- 4×4 棋盘，8 对 emoji 图案，洗牌随机
- 计时、计步（回合数：翻两张计 1 回合）
- 3D 翻转动画 + 匹配轻微缩放动效
- 防连点/输入锁：同一时间最多翻 2 张
- 胜利弹窗：展示本局用时/回合数/得分增量
- 用户系统：仅用户名登录（无需密码）+ 游客模式
- 成绩提交与排行榜：Top10（按总积分优先，同分按最佳回合/最佳用时）
- 分享链接：复制当前站点链接（可扩展为携带用户名参数）

**不在本期范围**
- 账号密码/第三方登录
- 多难度、主题皮肤、音效
- 永久存储与跨设备稳定排行（当前为演示用内存存储）

## 3. 用户与使用场景

**用户类型**
- 游客玩家：想快速玩一局，不关心留存数据
- 轻度竞争玩家：用固定用户名反复挑战，争取更高排名

**关键场景**
- 打开页面 → 游客进入 → 通关 → 再来一局
- 输入用户名登录 → 通关自动提交 → 查看排行榜

## 4. 用户体验与信息架构

**页面/视图**
- 游戏页（默认）
  - 统计区：回合、计时、进度
  - 控制区：重新开始、分享链接
  - 个人区：总积分、最佳回合、最佳用时
  - 棋盘：4×4 卡片
  - 弹窗：登录弹窗、胜利弹窗
- 排行榜页
  - Top10 表格：用户名、总积分、最佳回合、最佳用时

**核心交互规则**
- 点击卡片翻转；同一时间最多翻 2 张
- 第二张翻开后进入结算：
  - 相同：两张标记匹配并保持翻开
  - 不同：短暂停留后自动翻回
- 结算期间锁输入；已翻开/已匹配卡不可重复点击

## 5. 功能需求（详细）

### 5.1 游戏初始化
- 默认棋盘：4×4（16 张），8 对
- 卡面：emoji（从预置列表抽取 8 个 → 复制成对 → 洗牌）
- 重新开始：重置计时/回合/状态机并重新洗牌

### 5.2 计时与计步
- 计时：首张有效翻牌开始计时；胜利停止
- 计步口径：每翻两张计 1 回合（turn）
- UI 实时展示 `回合` 与 `用时`

### 5.3 胜利结算
- 全部匹配后弹窗：
  - 本局回合数
  - 本局用时
  - 本局得分增量（登录用户才会展示，游客为 `-`）
- 按钮：再来一局、关闭

### 5.4 用户系统（无密码）
- 登录入口：游戏开始前弹窗
- 用户名规则：1–16 位；允许中文、英文、数字、`_`、`-`
- 游客模式：可游玩但不提交排行
- 切换用户：回到游客并清空当前局状态

### 5.5 积分规则
- 基础分 100
- 回合加分：回合越少加分越多
- 用时加分：用时越短加分越多
- 服务端计算得分增量，前端仅提交回合/用时

### 5.6 排行榜
- 展示 Top10
- 排序：
  1) 总积分（降序）
  2) 最佳回合（升序，空值靠后）
  3) 最佳用时（升序，空值靠后）

### 5.7 分享
- 点击复制当前站点链接
- 失败时在 Toast 中显示链接

## 6. 后端需求（Express / Vercel Functions）

### 6.1 API 列表
- `GET /api/health`：健康检查
- `GET /api/config`：游戏配置
- `POST /api/users/login`：用户名登录（空为游客）
- `GET /api/users/:username`：获取用户数据
- `POST /api/scores`：提交成绩（服务端算分并更新用户/排行）
- `GET /api/leaderboard?limit=10`：获取排行榜

### 6.2 数据存储
- MVP：内存存储（适合演示）
- 风险：部署在 Vercel 上可能因冷启动/扩缩容导致数据重置
- 后续增强：切换 Supabase/Postgres 等永久存储

## 7. 非功能需求

- 性能：点击到翻转反馈 < 100ms；结算动画流畅（60fps 目标）
- 兼容：现代浏览器（Chrome/Safari/Edge）移动端与桌面端
- 安全：不在前端存放任何敏感密钥；后端不记录隐私数据
- 可维护性：前后端职责清晰，API 错误可被前端正确提示

## 8. 验收标准（Definition of Done）

- 任意合法用户名（如 `11`、`Alice`、`张三`、`a_b-1`）可登录成功
- 非法用户名（含空格/特殊符号）明确提示“用户名不合法”
- 游客可进入并完成一局，不触发成绩提交
- 登录用户通关后自动提交成绩并能在排行榜看到更新
- Vercel 线上 `GET /api/health`、登录与成绩提交接口可用
